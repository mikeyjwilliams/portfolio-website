UI-Challenge Great Idea Project

This was my first Sprint challenge at Lambda Schools. It was semi difficult at the time. But I have learned a lot from this project, and from the experiences since it.

  1. Read the Documentation carefully.
  2. Make sure you run through the MVP(Minimal Viable Product) specs before moving to anything else.
  3. Do not let yourself get stuck for more then 20 minutes...without going through documentation, research, or asking for help.
  4. Make sure you understand what is expected from the get go, if something is not clear ask for specification. This will save everyone headache and time.

With this project, we were mostly focused on HTML semantics, CSS fundamentals, Flexbox, and learning the GIT flow process of Lambda Schools. I think this project was a good start to understanding semantic design(without media queries). Also, a great kick off to flexbox and getting use to designing with it while building upon specificity in CSS and the cascade.
